# Proposal: 仪表板导出 PDF 功能

## 1. 背景

用户需要将仪表板导出为 PDF 文件，用于：
- 周报/月报生成
- 离线查看
- 分享给团队成员

## 2. Scope

### In Scope
- 导出当前仪表板为 PDF
- 支持导出当前视图或全尺寸
- 添加导出按钮到仪表板页面
- 支持 A4 / Letter 纸张大小

### Out of Scope
- 定时自动导出（Phase 2）
- 邮件发送 PDF（Phase 2）
- 导出历史记录

## 3. 用户场景

### 场景 1: 一键导出
- 用户在仪表板页面
- 点击「导出 PDF」按钮
- 系统生成 PDF 并下载

### 场景 2: 分享报告
- 用户需要将当前仪表板发给同事
- 导出 PDF 后通过微信/邮件发送

## 4. 技术方案

### 方案 A: 纯前端（推荐）
使用 `html2canvas` + `jspdf` 在浏览器端生成 PDF

**优点**：
- 无需后端
- 响应速度快
- 减少服务器负载

**缺点**：
- 大型 Dashboard 可能性能较差

### 方案 B: 后端渲染
使用 Puppeteer/Playwright 在服务端渲染

**优点**：
- 渲染质量高

**缺点**：
- 需要额外服务
- 部署复杂

**推荐方案 A**

## 5. 依赖包

```json
{
  "html2canvas": "^1.4.1",
  "jspdf": "^2.5.1"
}
```

## 6. 验收标准

- [ ] 点击导出按钮生成 PDF
- [ ] PDF 包含仪表板标题和图表
- [ ] 支持 A4 / Letter 大小
- [ ] 文件名格式：`{仪表板标题}_{日期}.pdf`
- [ ] 加载状态提示
- [ ] 错误处理（导出失败提示）
