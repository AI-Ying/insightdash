# Exploration: 工厂质量分析模块

## 用户痛点

工厂质量工程师/厂长面临的挑战：
- 不知道今天的良品率是多少
- 问题发生后才被动发现
- 不知道哪个工序/设备/班组问题最多
- 质量数据分散，难以汇总分析
- 报表靠人工整理，耗时

## 核心价值

**从"数据展示"到"发现问题"**

## 目标用户

| 角色 | 需求 |
|------|------|
| 厂长/车间主任 | 一眼看今日良品率，异常立即告警 |
| 质量工程师 | 分析问题根因，追踪改善措施 |
| 产线工人 | 知道自己工站的良品率 |

### 模块 2: 质量数据采集

**数据来源**：
| 来源 | 数据 | 格式 |
|------|------|------|
| PLC/传感器 | 尺寸、温度、压力 | MQTT/OPC UA |
| 人工录入 | 不良品原因、外观检测 | App/网页 |
| MES系统 | 订单、工序、批次 | API/CSV |

**最低可行方案**：
- CSV 上传 + API 数据源（已实现）
- 手工录入不良品表单

### 模块 3: 告警规则

**规则示例**：
| 规则 | 阈值 | 动作 |
|------|------|------|
| 良品率低 | < 95% | 微信通知 |
| 不良品突增 | 30min内 > 10件 | 告警 |
| 特定设备故障 | 连续3件不良 | 停线提醒 |

### 模块 4: 根因分析

**分析维度**：
- 时间维度：小时/班次/日/周/月
- 空间维度：工位/设备/班组/批次
- 维度组合：工位+班次、设备+时间段

**分析方法**：
- 柏拉图（80/20法则）
- 趋势对比（同环比）
- 散点图（参数关联）

## 数据模型建议

```typescript
// 质量数据
interface QualityRecord {
  id: string;
  timestamp: Date;
  workstation: string;      // 工位
  device: string;          // 设备
  shift: string;          // 班次
  total: number;          // 总产出
  good: number;           // 良品
  defect: number;         // 不良品
  defectType: string;     // 不良类型
  defectReason?: string;   // 不良原因
  operator?: string;       // 操作员
  batch?: string;         // 批次号

  "组装错误",
  "材料问题",
  "其他"
] as const;
```

## 实现路径

### Phase 1: MVP（1-2天）

**功能**：质量数据导入 + 基础看板

1. 创建质量数据 CSV 模板
2. 上传数据，生成看板
3. 4个基础图表：良品率、趋势、分布、工序排名

### Phase 2: 增强（3-5天）

**功能**：手工录入 + 告警

1. 不良品录入表单
2. 告警规则配置
3. 微信/邮件通知

### Phase 3: 完整版（1-2周）

**功能**：实时数据 + 分析

1. PLC/传感器数据接入
2. 自动刷新看板
3. 根因分析工具
4. 报告生成

## 竞争分析

| 产品 | 特点 | InsightDash 机会 |
|------|------|----------------|
| 工业大脑 | 大厂、复杂、贵 | 轻量、便宜、快速上手 |
| MES系统 | 功能全、贵 | 专注质量分析单项 |
| Excel | 灵活、普及 | 可视化、协作 |

## 推荐方向

**Phase 1 做"轻量质量看板"**：

- 用 CSV/API 导入数据
- 自动生成质量图表
- 一键分享给厂长
- 极低成本，快速验证
